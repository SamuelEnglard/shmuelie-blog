Ryan Lange has a great post about installing XNA in Visual Studio 2012 (and even one for getting the help offline). While doing it I made some tweaks and had some notes on things you should know while doing it.

  • You don’t actually need to install Visual Studio 2010 with XNA on the machine that is going to have 2012 on it. You just need a machine that has/had it, a program that can extract files from executables/MSIs (like zip), and the WP7.1 SDK in ISO form.
    1) Copy the extension files from the machine that has/had 2010 with XNA installed as told said in step 1.
    2) Open the ISO and navigate to “WCU\WindowsPhone”. In there is a file called “XNAGS40_setup.exe”. Sadly you can’t just run this. Extract it to a folder on your hard drive.
    3) In the resulting files list there is a file called “redists.exe”. Sadly you can’t just run this either, you must extract it too.
    4) Rename the files with “XML” in their name to have the extension “xml”, the ELUA with “rtf”, and the remaining files with “msi”. Once done run each MSI, and if there is an error ignore it.
  • Note that that running
    “C:\Program Files (x86)\Microsoft Visual Studio 11.0\Common7\IDE\devenv.exe” /setup

    can take some time, so watch it using something like Task Manager to see when it’s done.

EDIT 12/17/2013 the trick still works in VS 2013, just have to make sure you change “extension.vsixmanifest” to have version 12.0 instead of 11.0 as the blog post above says.
