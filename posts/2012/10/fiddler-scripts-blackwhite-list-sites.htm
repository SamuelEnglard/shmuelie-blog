When using Fiddler to debug a website one problem I have is that there are just too many sessions. Even if the only web connection your computer is making is via the browser to the site youâ€™re testing odds are that the website will make three thousand calls to third-party sites (or at least it feels that way). So of course I created a script to fix this problem. There are two version of the script. The first version creates a black list, useful for when you just want to remove sessions from one annoying third-party but want 90% of the traffic still. If though you only care about traffic from one site you can use the white list version instead. Both version use ValidUrlOrHostname method from the first post to decide if they should block a session.

Black List

The following code shows a text prompt that asks for a comma separated list of domains to block. 

   1: public static var sHideSites: String = "";
   2: public static ToolsAction("Hide Sites")
   3: function HideSites() {
   4:     sHideSites = FiddlerObject.prompt("Enter host name to hide\nEmpty text will clear list\nTo include sub domains, start with *.\nSeperate domains by ','", sHideSites);
   5: }


Then in the OnBeforeRequest method you put the following




   1: if ((sHideSites.Length != 0) && ValidHost(oSession.hostname, sHideSites))
   2: {
   3:     oSession["ui-hide"] = "do not care.";
   4: }


White List


The white list uses almost identical code. The difference are that it blocks unless the site is valid and uses different variable names (so that you can easily use both)


The UI code



   1: public static var sShowOnlySites: String = "";
   2: public static ToolsAction("Show Only Sites")
   3: function ShowOnlySites() {
   4:     sShowOnlySites = FiddlerObject.prompt("Enter host name to only show\nEmpty text will clear list\nTo include sub domains, start with *.\nSeperate domains by ','", sShowOnlySites);
   5: }

Then in the OnBeforeRequest method you put the following



   1: if ((sShowOnlySites.Length > 0) && (!ValidHost(oSession.hostname, sShowOnlySites)))
   2: {
   3:     oSession["ui-hide"] = "do not care.";
   4: }
